# IMPLEMENT_INTERFACE

## Purpose
Implement a scoped frontend change, interaction flow, or client integration.

## When to Use
- The assignment is primarily UI or client-side behavior

## Inputs
- Objective
- Relevant components and routes
- Constraints

## Constraints
- Stay inside frontend ownership
- Do not invent backend behavior

## Process
1. Read the relevant UI files and interfaces.
2. Implement the requested client-side behavior.
3. Report changed files, UX findings, and blockers.

## Output
A structured specialist result for frontend work.

## Failure Modes
- Missing API contract: request evidence or escalate
- Brand ambiguity: recommend `signifer`

## Example
`artifex::IMPLEMENT_INTERFACE` to wire a settings form to an existing API.
